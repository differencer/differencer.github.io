提交的时候发现报错：

20:42:28.043: [xuecheng-plus] git -c credential.helper= -c core.quotepath=false -c log.showSignature=false push --progress --porcelain origin refs/heads/main:refs/heads/main --set-upstream --tags
remote: No anonymous write access.
fatal: Authentication failed for 'https://github.com/differencer/xuecheng-plus.git/'

解决方案：

方案1、在git ide 中或者 idea的终端中 输入以下命令

git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

提交又报错了。。

20:55:13.214: [xuecheng-plus] git -c credential.helper= -c core.quotepath=false -c log.showSignature=false push --progress --porcelain origin refs/heads/main:refs/heads/main --set-upstream --tags
remote: No anonymous write access.
fatal: Authentication failed for 'https://github.com/differencer/xuecheng-plus.git/'

解决方案：

方案1、 挂的梯子不稳定，换一个稳定的，延时短节点，

方案2、如果远程仓库 URL 是 https://github.com/differencer/xuecheng-plus.git，可以使用以下命令将其更新为 SSH 格式：

git remote set-url origin git@github.com:differencer/xuecheng-plus.git

SSH 使用密钥进行身份验证，通常比 HTTPS 更稳定。

捏妈妈爹，终于成功了

下边创建开发分支。新建一个分支，点击IDEA右下角的分支标识：

基础工程搭建

工程结构关系

项目可以分为前端工程和后端工程，这里主要介绍后端工程的结构关系

后端使用 Maven 来进行项目的管理和构建。整个项目分为三大类工程：父工程、基础工程 和微服务工程。

父工程：对依赖包的版本进行管理 、本身为Pom工程，对子工程进行聚合管理

基础工程：继承父类工程 提供基础类库 提供工具类库

微服务工程： 分业务分模块实现功能，依赖基础工程，间接继承父工程

构建父工程

Project Structure,首先检查jdk是否配置正确，并进行配置。

注意我们用的时1.8的jdk

进入Modules界面，新建模块

进入新建模块界面，选择Spring Initializr，填写模块的信息。

注意：这里Server URL默认是start.spring.io，如果连接不上可换为start.aliyun.com（建议）。

这一步太容易错了，仔细看看这些玩意的配置。

之后接默认连点

ok现在就在总工程的目录下新建了一个 新的工程，将这个工程作为整个工程的父工程，用于规定所使用的包，类的版本

里面的文件除了pom文件一个不留

父工程的作用，主要时对依赖的版本进行管理，即父工程中没有代码，不用去依赖其它的包，它的作用是限定其它子工程依赖包的版本号，即在dependencyManagement 中去编辑即可

  • 编辑依赖的包的版本号、打包插件等。

    这里具体代码我就不粘贴了，只介绍简单介绍以下

<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 https://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>
    <groupId>com.xuecheng</groupId>
    <artifactId>xuecheng-plus-parent</artifactId>
    <version>0.0.1-SNAPSHOT</version>
    <name>xuecheng-plus-parent</name>
    <description>xuecheng-plus-parent</description>
    <packaging>pom</packaging>
    
  <properties>
        <java.version>1.8</java.version>
        <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
        <!--此处规定了一些，变量，包的版本，这一块有点像宏定义-->
  </properties>

</dependencyManagement> 
      <!--依赖管理部分-->
  <build>
        <finalName>${project.name}</finalName>
        <!--编译打包过虑配置-->
        <resources>
            <resource>
				<!--略-->
            </resource>
		</resources>
        <plugins>
            <plugin>
				<!--略-->
            </plugin>
        </plugins>
    </build>

</project>

更改完毕后，将pom文件右键，添加为maven工程，这样次啊能被maven管理

之后上传代码到自己的分支。

构建基础工程

基础工程的职责是提供一些系统架构所需要的基础类库以及一此工具类库。

同上面步骤：

在新建的 工程中的pom文件中复制资料中的依赖库，指的注意的是，整个基础工程是需要继承父工程的！！！

提别体现在

<parent>
    <groupId>com.xuecheng</groupId>
    <artifactId>xuecheng-plus-parent</artifactId>
    <version>0.0.1-SNAPSHOT</version>
    <relativePath>../xuecheng-plus-parent</relativePath>
</parent>

更改后，提交代码到git
